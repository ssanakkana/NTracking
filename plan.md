## 1. 项目概述

### 1.1 项目背景
随着人工智能技术的快速发展，"AI Native OS"概念逐渐兴起。传统的操作系统被动响应用户指令，而 AI Native OS 具备**情境感知**能力，能够主动理解用户行为、意图，并提供智能化辅助。

本项目的核心是开发一个**用户行为管理器**，作为 AI Native OS 的基础设施，实现对用户计算机使用行为的智能监控、记录与分析。

### 1.2 项目愿景
> **让操作系统理解用户，而非仅仅执行命令**

通过追踪用户的键盘、鼠标输入，应用使用情况，以及网页/H5应用交互行为，构建用户的"数字记忆"，为 AI 助手提供上下文感知能力，实现：
- 自动总结用户工作/学习内容
- 智能检索历史行为
- 主动提供个性化建议
- 构建个人知识图谱

### 1.3 项目定位
- **性质：** 个人实验项目（非商业用途）
- **平台：** Windows 10/11
- **用户：** 开发者本人
- **目标：** 验证 AI Native OS 核心概念，探索人机交互新范式

## 2. 项目目标

### 2.1 核心目标

| 目标 | 描述 | 衡量标准 |
|------|------|----------|
| **行为追踪** | 实时记录用户在电脑上的所有操作 | 准确率 > 95% |
| **隐私保护** | 所有数据本地存储，加密处理 | 零数据泄露 |
| **性能优化** | 系统资源占用 < 5% CPU，< 500MB 内存 | 不影响正常使用 |
| **智能总结** | 自动生成时间段行为报告 | 总结准确率 > 85% |
| **语义检索** | 支持自然语言查询历史行为 | 查询响应 < 2秒 |

### 2.2 技术目标

1. **无侵入监控：** 使用 UI Automation 等官方接口，避免底层 Hook
2. **模块化设计：** 各功能模块解耦，便于扩展和维护
3. **本地 AI 集成：** 集成本地 LLM，无需云端依赖
4. **杀软友好：** 通过代码签名和合规设计，避免被误杀

### 2.3 实验目标

1. 验证 UI Automation 在 Electron 应用中的可行性
2. 探索本地 LLM 在行为摘要中的应用效果
3. 评估向量数据库在个人记忆检索中的性能
4. 总结隐私保护与功能实现的平衡点


## 4. 功能模块

### 4.1 模块清单

#### **M1 - 窗口监控模块**
- **功能：** 实时追踪当前活动窗口
- **技术：** `GetForegroundWindow` + `UI Automation`
- **数据：** 窗口标题、进程名、窗口类名、时间戳
- **优先级：** P0 (核心)

#### **M2 - 进程监控模块**
- **功能：** 记录应用启动/关闭事件
- **技术：** WMI `Win32_Process` 事件监听
- **数据：** 进程名、路径、启动时间、使用时长
- **优先级：** P0 (核心)

#### **M3 - 文本输入监控模块**
- **功能：** 获取输入框内容（文档、搜索框等）
- **技术：** `UI Automation` ValuePattern
- **数据：** 控件类型、输入内容（脱敏）、应用上下文
- **优先级：** P0 (核心)

#### **M4 - 浏览器监控模块**
- **功能：** 获取 URL、标签页标题、书签
- **技术：** 浏览器扩展 + Native Messaging
- **数据：** URL、页面标题、访问时间、停留时长
- **优先级：** P0 (核心)

#### **M5 - Electron 应用适配模块**
- **功能：** 监控 Bilibili、抖音、VS Code 等 Electron 应用
- **技术：** `UI Automation` + 窗口标题解析
- **数据：** 视频标题、UP主、播放进度（Bilibili）
- **优先级：** P0 (核心)

#### **M6 - 屏幕截图模块**
- **功能：** 定时/事件触发截图（OCR 兜底）
- **技术：** `Graphics.CopyFromScreen` + 按需触发
- **数据：** 截图文件路径、OCR 识别文本
- **优先级：** P1 (增强)
- **策略：** 仅在 UIA 无法获取内容时使用

#### **M7 - 事件存储模块**
- **功能：** 结构化存储所有行为事件
- **技术：** SQLite (Microsoft.Data.Sqlite)
- **数据表：** 
  - `Events` (事件表)
  - `Applications` (应用表)
  - `Windows` (窗口表)
  - `Inputs` (输入表)
  - `Summaries` (摘要表)
- **优先级：** P0 (核心)

#### **M8 - 数据加密模块**
- **功能：** 本地数据加密存储
- **技术：** Windows DPAPI
- **范围：** 所有数据库文件、配置文件
- **优先级：** P0 (核心)

#### **M9 - 本地 LLM 模块**
- **功能：** 行为摘要、意图理解
- **技术：** llama.cpp + LLamaSharp
- **模型：** Qwen2.5-1.5B-Instruct (量化版)
- **任务：**
  - 每小时行为摘要
  - 每日工作总结
  - 意图分类
- **优先级：** P0 (核心)

#### **M10 - Embedding 模块**
- **功能：** 文本向量化
- **技术：** bge-small-zh-v1.5 (ONNX)
- **用途：** 语义检索、相似行为匹配
- **优先级：** P1 (增强)

#### **M11 - 定时总结模块**
- **功能：** 自动生成行为报告
- **技术：** 定时任务 + LLM Prompt
- **频率：** 每小时、每天、每周
- **输出：** 文本摘要、时间分配统计
- **优先级：** P0 (核心)

#### **M12 - 语义检索模块**
- **功能：** 自然语言查询历史行为
- **技术：** RAG + 向量搜索
- **示例：** "我上周看了什么视频？"
- **优先级：** P1 (增强)

#### **M13 - 系统托盘模块**
- **功能：** 显示运行状态、快速开关
- **技术：** WPF NotifyIcon
- **功能：**
  - 显示"正在记录"状态
  - 一键暂停/恢复
  - 快速查看今日摘要
- **优先级：** P0 (核心)

#### **M14 - 时间线视图模块**
- **功能：** 可视化展示行为历史
- **技术：** WPF Timeline 控件
- **展示：** 时间轴、应用使用分布、活动热力图
- **优先级：** P1 (增强)

#### **M15 - 设置面板模块**
- **功能：** 隐私配置、白名单、数据管理
- **技术：** WPF Settings Window
- **配置项：**
  - 敏感应用白名单
  - 数据保留时长
  - 截图频率
  - AI 模型选择
- **优先级：** P0 (核心)

#### **M16 - 敏感应用过滤模块**
- **功能：** 自动忽略密码管理器、银行应用
- **技术：** 进程名白名单匹配
- **默认列表：** 1Password, Bitwarden, 银行客户端等
- **优先级：** P0 (核心)

#### **M17 - 隐私指示器模块**
- **功能：** 明确显示"正在记录"状态
- **技术：** 托盘图标颜色/动画
- **设计原则：** 用户始终知晓监控状态
- **优先级：** P0 (核心)

#### **M18 - 数据导出模块**
- **功能：** 用户可导出/删除所有数据
- **格式：** JSON, CSV
- **优先级：** P1 (增强)

## 7. 风险评估

### 7.1 技术风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| **UI Automation 覆盖率不足** | 中 | 高 | 准备 OCR 兜底方案；优先使用官方接口 |
| **本地 LLM 性能不足** | 中 | 中 | 使用量化模型；支持 CPU/GPU 切换；异步处理 |
| **杀毒软件误报** | 高 | 高 | 使用代码签名；避免底层 Hook；加入白名单 |
| **Electron 应用更新导致失效** | 中 | 中 | 模块化设计；快速适配机制 |
| **数据库性能瓶颈** | 低 | 中 | 定期清理；索引优化；分表策略 |

### 7.2 隐私风险

## 8. 隐私与合规

### 8.1 设计原则

1. **Privacy by Design（隐私设计）**
   - 默认不记录敏感信息
   - 数据本地存储，不上云
   - 用户完全掌控数据

2. **Transparency（透明性）**
   - 明确告知用户记录内容
   - 实时显示监控状态
   - 提供数据查看入口

3. **User Control（用户控制）**
   - 一键暂停/恢复
   - 可删除任意时间段数据
   - 可导出全部数据

4. **Data Minimization（数据最小化）**
   - 仅记录必要信息
   - 定期自动清理
   - 不记录密码等敏感内容

### 8.2 数据收集范围

#### ✅ **会收集的数据**
- 应用名称、窗口标题
- 网页 URL、页面标题
- 输入框文本（非密码框）
- 时间戳、使用时长
- 鼠标点击的控件名称（非坐标）

#### ❌ **不会收集的数据**
- 密码框内容
- 银行/支付应用数据
- 加密通讯内容（如 Signal）
- 系统凭据
- 生物识别信息

### 8.3 数据存储策略

| 数据类型 | 存储位置 | 加密 | 保留时长 |
|----------|----------|------|----------|
| **行为事件** | SQLite | DPAPI | 可配置（默认 90 天） |
| **向量索引** | SQLite-vec | DPAPI | 与事件同步 |
| **截图文件** | 本地文件系统 | DPAPI | 可配置（默认 7 天） |
| **AI 摘要** | SQLite | DPAPI | 永久（用户可删） |
| **配置文件** | AppData | DPAPI | 永久 |

### 8.4 合规声明

本项目严格遵守：
- **《中华人民共和国个人信息保护法》**
- **《中华人民共和国网络安全法》**
- **GDPR（参考原则）**

**使用限制：**
- 仅限个人学习实验使用
- 禁止用于商业目的
- 禁止在未经他人同意的情况下监控他人
- 禁止用于非法用途
